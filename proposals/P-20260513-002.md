# P-20260513-002 little-garden 测试体系建设

## 目标
为 little-garden 项目建立测试体系：QUnit 单元测试 + Playwright E2E 测试

## 技术方案

### A. QUnit 单元测试
- 框架：QUnit 3.x（单文件，零依赖，浏览器直接运行）
- 测试文件：`tests/qunit/` 目录
- 运行方式：浏览器打开 `tests/qunit/index.html` 或 Node.js CLI
- 测试范围：
  - Button/Text/ProgressBar 等 UIElement 子类
  - Scene 基类（changeScene、render cycle）
  - Game 类核心方法
  - 学习模块逻辑（MathGardenScene、LearningScene 答题逻辑）
  - 输入处理（InputManager click/touch）
  - Tween/ParticleSystem 动画系统

### B. Playwright E2E 测试
- 框架：@playwright/test
- 测试文件：`tests/e2e/` 目录
- 测试项：
  - 游戏加载无崩溃
  - 主菜单渲染
  - 花园场景 UI 元素可见
  - 学习花园按钮点击 → 场景切换
  - 数学花园关卡列表渲染
  - Canvas 无 JS Error

## 文件结构
```
tests/
├── qunit/
│   ├── index.html          # QUnit 运行器
│   ├── ui-element.test.js  # UI 组件测试
│   ├── scene.test.js       # 场景测试
│   └── learning.test.js    # 学习模块测试
└── e2e/
    ├── playwright.config.js
    └── garden.spec.js      # E2E 测试
```

## 验收标准
- QUnit: 所有测试通过，覆盖核心游戏逻辑
- Playwright: 游戏加载无崩溃，UI 元素可交互
- GitHub Actions: CI 中运行测试
